Hi Derek,

When I did the data mapping I noticed that some entries looked like they were entered manually but they also had concept codes. I went back and looked at the histopathology table in DATOP and saw that we got those entries from there.

 

This is the history behind it:

  1. When we started caMOD we did not have a disease vocabulary and disease entries were hand-typed by the submitting scientists.
  2. After we got the vocabulary we entered only the concept codes in the EVS_id column in the histopathology table. No entries were put in the tumorclassification column.
  3. We identified all old (prior to the vocabulary) entries with 000000 in the EVS_id column and keeping the hand-typed entry in the tumorclassification column.
  4. We used the same system for manually entered entries when the users could not find the desired vocabulary term in the vocabulary tree.
  5. Later we tried to map the 000000 entries to the vocabulary.
  6. Starting with caMOD 2.0 we now enter the concept code and the name in the disease table for items selected from the vocabulary tree. We still employ the 000000 system for manually entered data.

 

These mappings that we attempted according to #6 contain errors and in general do not look good to me anymore. I would like to reverse these mappings by replacing the concept codes with 000000.

The attached table contains the disease table with corrected entries. Some entries contained 000000 in the concept code column but no entry in the name field. For these entries, I added “not specified” in the name field.
