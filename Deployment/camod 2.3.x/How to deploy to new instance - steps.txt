Building new instance for caMOD 2.3

Step 1.  /local/content/evstree/tree_cache_files/ holds all the EVSTree cache files - 
a. Create this folder structure on the new instance.
b. Move all the .index and .data files to /local/content/evstree/tree_cache_files/ on the specified tier.
c. Verify that this folder has read and write access from EVSTree application

Step 2.  /local/content/camod/config/ holds camod.properties and mail.properties - 
a. Create this folder structure on the new instance.
b. Move both the camod.properties and mail.properties to /local/content/camod/config on the specified tier

Step 3.  <jboss_home>/deploy holds the oracle-ds.xml and properties-service.xml files
a. Copy the oracle-ds.xml and properties-service.xml files from JBoss_home/deploy folder on the old instance to the new instance location
b. Comment out or remove the reference to the ApplicationSecurityConfig.xml (no longer used) 
For Example:
	<!--gov.nih.nci.security.configFile=/usr/local/jboss-4.0.2/server/csmconfig/ApplicationSecurityConfig.xml-->


Step 4.  /usr/local/jboss-4.0.2/server/ncicb-app/config holds the login-config.xml 
Move this file from previous location on specified tier to the location on the new instance

Note:  Verify that the working LDAP server is being referenced according to the LDAP assigned to the specified tier
For example:
    <!-- camod application -->
    <application-policy name = "camod">
       <authentication>
           <login-module code = "gov.nih.nci.security.authentication.loginmodules.LDAPLoginModule" flag = "required" >
               <module-option name = "ldapHost">ldaps://cbioweb-stage-ldap.nci.nih.gov:636</module-option>
               <module-option name="ldapSearchableBase">ou=nci,o=nih</module-option>
               <module-option name="ldapUserIdLabel">cn</module-option>
           </login-module>
       </authentication>
    </application-policy>
    
Step 6.  Add another entry in the apache conf files to redirect /EVSTree context to tomcat container.   

Step 7.  Build the camod.war and EVSTree.war to the <jboss_home>/deploy folder with specified tags
